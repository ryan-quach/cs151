# CS151 Final Project — LeNet-5 Hardware Accelerator

RTL implementation of a LeNet-5 neural network inference engine in Verilog, verified on EDA Playground (Icarus Verilog). Classifies MNIST handwritten digits (0–9) using trained int8 weights.

---

```

├── train_lenet5.py            # Train on MNIST, quantise to int8, export hex files
```

## Architecture

LeNet-5 with ReLU activations and max-pooling. All weights and activations use **8-bit signed integers**. After each layer's MAC accumulation, a 24-bit result is arithmetically right-shifted (RSHIFT) before the ReLU/saturation clamp.

```
image (32x32x1, int8)
  -> C1  Conv(1->6,   5x5) + ReLU  -> 28x28x6
  -> S2  MaxPool(2x2)               -> 14x14x6
  -> C3  Conv(6->16,  5x5) + ReLU  -> 10x10x16
  -> S4  MaxPool(2x2)               -> 5x5x16
  -> C5  Conv(16->120,5x5) + ReLU  -> 1x1x120
  -> F6  FC(120->84)  + ReLU
  -> OUT FC(84->10)   (raw logits, no softmax)
```

| Module | Description |
|--------|-------------|
| `mac_unit` | Registered 8-bit signed multiply-accumulate, 24-bit accumulator |
| `relu8` | Combinational saturating ReLU: 24-bit in → 8-bit out |
| `conv_layer` | Parameterised 2-D convolution + RSHIFT + ReLU |
| `maxpool_layer` | Parameterised 2×2 max-pooling, stride 2 |
| `fc_layer` | Fully-connected + RSHIFT + ReLU or signed saturation |
| `lenet_top` | 16-state FSM sequencing all seven layers |

## Setup

### 1. Install Python dependencies

```bash
pip install torch torchvision numpy
```

### 2. Train the model and generate weights

```bash
python train_lenet5.py
```

This trains LeNet-5 on MNIST for 5 epochs (~98% accuracy), quantises all weights to int8, and writes the `weights/` folder. At the end it prints the RSHIFT values you need for the next step:

```
Paste these RSHIFT values into lenet_top in lenet.v:
    localparam RSHIFT_C1  = 9;
    localparam RSHIFT_C3  = 10;
    ...
Integer simulation prediction: 7  (ground truth: 7)
Simulation CORRECT
```

### 3. Update RSHIFT values in lenet.v

Open `lenet.v` and find the block near the bottom of `lenet_top`:

```verilog
// ── RSHIFT values (update from train_lenet5.py output) ──────
localparam RSHIFT_C1  = 9;
localparam RSHIFT_C3  = 10;
localparam RSHIFT_C5  = 10;
localparam RSHIFT_F6  = 10;
localparam RSHIFT_OUT = 9;
```

Replace with the values printed by `train_lenet5.py`. Do the same in `lenet_parallel.v` if using the parallel design.

### 4. Note the ground-truth digit

Check `weights/test_image_label.txt` for the digit class of the exported test image (e.g. `7`).

## Running in EDA Playground

1. Go to [edaplayground.com](https://edaplayground.com) and set the simulator to **Icarus Verilog**.
2. Paste `lenet.v` (or `lenet_parallel.v`) into the **Design** pane.
3. Paste `tb_lenet.v` into the **Testbench** pane.
4. Click **+** above the left pane and upload all 12 `.hex` files from `weights/`:
   - `c1_w.hex`, `c1_b.hex`
   - `c3_w.hex`, `c3_b.hex`
   - `c5_w_0.hex`, `c5_w_1.hex`, `c5_b.hex` *(c5 is split into two files)*
   - `f6_w.hex`, `f6_b.hex`
   - `out_w.hex`, `out_b.hex`
   - `test_image.hex`
5. In `tb_lenet.v`, set the two parameters at the top:
   ```verilog
   parameter TRAINED_WEIGHTS = 1;
   parameter EXPECTED_CLASS  = 7;  // replace with value from test_image_label.txt
   ```
6. Click **Run**.

## Extra Credit — Performance Analysis

### Parallelism (lenet_parallel.v)

`lenet_parallel.v` adds a `PARALLEL` parameter that computes P output channels simultaneously. At maximum parallelism (P = OUT_CH per layer):

| Layer | Sequential cycles | Parallel cycles | Speedup |
|-------|-----------------|----------------|---------|
| C1 (P=6) | 117,600 | 19,600 | 6× |
| C3 (P=16) | 240,000 | 15,000 | 16× |
| C5 (P=120) | 48,000 | 400 | 120× |
| F6 (P=84) | 10,080 | 120 | 84× |
| OUT (P=10) | 840 | 84 | 10× |
| **Total** | **418,096** | **35,425** | **11.8×** |

### Hardware vs GPU Comparison

Benchmarked using `benchmark.py` on Google Colab (Tesla T4 GPU):

```bash
python benchmark.py
```

| Platform | Latency (batch=1) | Peak Throughput | Power |
|----------|------------------|----------------|-------|
| CPU (PyTorch float32) | 660 µs | 13,453 img/s | ~15–45 W |
| GPU (Tesla T4) | 417 µs | 688,913 img/s | 70 W |
| FPGA sequential | 4,181 µs | 239 img/s | ~0.3 W |
| FPGA parallel | **354 µs** | 2,823 img/s | ~0.5 W |

The FPGA parallel design achieves lower single-image latency than both CPU and GPU. The GPU only pulls ahead at large batch sizes. See `analysis.md` for the full discussion.

## How RSHIFT Works

int8 weights are scaled to fill [-127, 127] (typical scale factor: 200–400×). After a full layer accumulation, the 24-bit result is far outside the [0, 127] output range — without correction, every output saturates to ±127 regardless of the weights.

`train_lenet5.py` solves this by running an exact integer forward-pass simulation that mirrors the RTL arithmetic, measuring the maximum accumulator value at each layer, and computing the minimum right-shift needed to bring it into range. This value is printed and must be pasted into `lenet_top`.
